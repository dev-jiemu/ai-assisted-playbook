지금까지 수정한 내용을 바탕으로, 팀원들이 한눈에 파악할 수 있도록 GitHub [Pull Request / Issue] 본문 템플릿에 맞춰 내용을 채워줘. 

아래 양식을 채워서 마크다운 코드로 출력해 줘:

## 📝 개요 (Summary)
- 이번 작업의 목적과 변경 사항 요약

## 🐛 관련 이슈 및 문제점 (Context & Problem)
- 어떤 에러나 요구사항 때문에 시작된 작업인지 기술 (논의된 로그/현상 중심)

## 🛠️ 수정 및 구현 내용 (Changes)
- [ ] 핵심 수정 사항 1 (관련 파일명 기재)
- [ ] 핵심 수정 사항 2

## 💻 핵심 코드 변경 사항 (Key Code Snippets)
- 가장 중요한 변경 로직 코드를 `diff` 형태로 표현해 주거나 최종 코드로 작성해 줘.

## 🚨 리뷰어 참고 사항 (Notes for Reviewers)
- 테스트 시 주의해야 할 점이나, 외부 레포지토리 연동 시 신경 써야 할 부분 기술
